# BUG-001: Daniel 3 Mapping and Multi-Verse Span Parsing

## Status
**Proposed Fix Available** (See `BUG-001.diff`)

## Problem Description

### 1. Multi-Verse Span Parsing ("The Descendant Bug")
Bible Gateway often bundles multiple verses within a single high-level `<span>` tag in translations like `RSVCE` or `DRA`.
- **Original Behavior:** The parser used `.find_all(['span'])`, identifying only the first verse marker in a block. `get_text()` then merged all subsequent verses into that first one, breaking verse-level filtering and causing missing verses in ranges.
- **Root Cause:** DOM-based parsing that didn't account for linear sequence of text nodes across different child elements within a parent span.

### 2. Daniel 3 Numbering Mismatch (Catholic Lectionary)
The USCCB lectionary follows `NABRE` numbering, where the **Prayer of Azariah** is Daniel 3:24–90. In `RSVCE`, these are Greek additions using internal numbering 1–67.
- **Original Behavior:** Requesting `Daniel 3:25` from the USCCB source in `RSVCE` returned the Hebrew Verse 25 (the King's reaction) instead of the Prayer of Azariah.
- **Root Cause:** Lack of a translation-aware mapping layer for book-specific numbering discrepancies.

### 3. Disjoint Verse Ranges
Lectionaries frequently request disjoint ranges (e.g., `Daniel 3:25, 34-43`).
- **Original Behavior:** The reference parser only handled simple contiguous ranges (`start-end`).
- **Root Cause:** Regex-based parsing was too restrictive for comma-separated or "and"-joined lists.

## Proposed Solution

1.  **Descendant Iteration:** Re-engineer `Parser.parse` to use `container.descendants`. This linear scan correctly identifies `versenum` and `chapternum` markers as they appear, allowing the parser to "break out" verses from within shared spans.
2.  **Mapping Engine:** Introduce `src/bgmd/mapping.py` to handle translation-specific numbering offsets (Daniel 3) and unify the existing Psalm mapping logic.
3.  **Full Chapter Fallback:** CLI now detects disjoint ranges and triggers a full chapter fetch, allowing the parser to surgically filter the specific verses requested from the complete text.
4.  **Word Spacing:** Aggressively add spaces between text nodes during extraction to prevent word-joining when HTML tags are stripped.

## Files Involved
- `src/bgmd/parser.py`: Updated extraction logic.
- `src/bgmd/cli.py`: Updated reference parsing and fetch logic.
- `src/bgmd/mapping.py`: New module for reference normalization.
- `src/bgmd/fetcher.py`: Updated to support raw search strings and improved caching.
